Core Platform Architecture

The Microsoft Azure Stack Powering Mysoft Neo

Built from the ground up on enterprise-grade Microsoft cloud infrastructure — ensuring enterprise customers get the performance, compliance, and scalability they demand.

.NET Core 8 · Azure App Services

Scalable .NET Core APIs

Mysoft Neo backend is a suite of independently deployable .NET Core 8 microservices hosted on Azure App Services with auto-scaling rules. Each service — Reservations, Billing, POS, Housekeeping — scales horizontally under peak load (check-in rushes, weekend F&B surges) without affecting adjacent services. Azure API Management provides a single, versioned gateway with rate limiting, OAuth 2.0 enforcement, and end-to-end TLS.

  • Horizontal scale-out via Azure App Service Plans
  • Background jobs via Azure WebJobs + Quartz.NET
  • Secure secrets via Azure Key Vault integration
  • CI/CD via Azure DevOps pipelines
Core Engine
Angular 17 · RxJS · SignalR

Real-Time Angular Frontend

The Mysoft Neo UI is a high-performance Angular 17 single-page application using OnPush change detection and lazy-loaded feature modules. The room allocation grid renders up to 500 rooms in a virtual scroll canvas with zero layout jank. Azure SignalR Service pushes real-time housekeeping status, reservation arrivals, and POS order states to every connected terminal simultaneously — no polling, no stale data.

  • Virtual scroll for 500+ room matrices
  • Real-time push via Azure SignalR Service
  • PWA-ready for tablet front desk usage
  • Role-based UI — Receptionist / Manager / Owner
Azure SQL · EF Core · Geo-Replication

Enterprise Data Integrity via Azure SQL

Every monetary transaction — room charges, F&B bills, banquet invoices, payroll journals — is persisted in Azure SQL Database with full ACID compliance. Row-level locking and snapshot isolation prevent dirty reads across simultaneous front-desk and dining-hall transactions. Active geo-replication ensures a readable secondary replica with <5s lag, and automated backups with point-in-time restore safeguard against data loss events.

  • ACID transactions across PMS + POS + Payroll
  • Active geo-replication for DR readiness
  • Tenant-isolated schemas for data sovereignty
  • Query performance insights via Azure Monitor

B2B Problem & Solution Map

From Fragmented Legacy to a Unified Nexus

Independent hotels and resorts operate on a patchwork of disconnected tools. Mysoft Neo collapses that complexity into a single, multi-tenant cloud hub.

Legacy Reality

The Fragmented Hotel Stack

  • Siloed Desktop Software

    Separate, incompatible applications for PMS, POS, and payroll — each requiring local server hardware and manual data transfer via USB or email.

  • Manual Night Audit (4–6 hours)

    A dedicated overnight accountant manually posts charges, reconciles tills, and generates reports — an error-prone, expensive operation repeated every single night.

  • Double-Booking Crises

    OTA platforms updated manually via extranets — slow, inconsistent, and a direct source of overbooking incidents and guest dissatisfaction.

  • Flat, Intuition-Based Pricing

    Room rates set weekly by a revenue manager using spreadsheets — missing real-time demand signals, losing thousands in potential RevPAR every high-demand day.

  • Zero Consolidated Reporting

    No single view of occupancy, F&B revenue, payroll cost, and banquet billing. Owners compile management accounts from multiple exports — days after month end.

Nexus
Mysoft Neo

The Unified Cloud Model

  • Single Multi-Tenant Cloud Platform

    Stay, dining, housekeeping, payroll, and banquet live in one Azure-hosted hub. No local servers, no VPNs, no manual sync — access from any browser, anywhere.

  • Autonomous Night Audit (~4 minutes)

    The Night Audit Engine runs automatically every night — posting charges, reconciling POS, rolling the date, and emailing reports. Zero staff. Zero errors.

  • Real-Time OTA Sync — <90 Seconds

    Every booking and cancellation propagates across all OTA channels in under 90 seconds via the dedicated Azure sync micro-service. Double bookings become structurally impossible.

  • Dynamic Tariff Engine — Always Optimised

    The Automated Revenue Manager continuously monitors occupancy and demand signals, updating rates across all segments and channels within defined guardrails — maximising every night's yield.

  • Unified Real-Time Dashboard

    Owners and GMs see a live consolidated view of occupancy, RevPAR, F&B revenue, material consumption, payroll cost, and banquet pipeline — updated by the second via Azure SignalR.
