Core Platform Architecture

The Microsoft Azure Stack Powering Mysoft Neo

Built from the ground up on enterprise-grade Microsoft cloud infrastructure — ensuring enterprise customers get the performance, compliance, and scalability they demand.

.NET Core 8 · Azure App Services

Scalable .NET Core APIs

Mysoft Neo backend is a suite of independently deployable .NET Core 8 microservices hosted on Azure App Services with auto-scaling rules. Each service — Reservations, Billing, POS, Housekeeping — scales horizontally under peak load (check-in rushes, weekend F&B surges) without affecting adjacent services. Azure API Management provides a single, versioned gateway with rate limiting, OAuth 2.0 enforcement, and end-to-end TLS.

  • Horizontal scale-out via Azure App Service Plans
  • Background jobs via Azure WebJobs + Quartz.NET
  • Secure secrets via Azure Key Vault integration
  • CI/CD via Azure DevOps pipelines
Core Engine
Angular 17 · RxJS · SignalR

Real-Time Angular Frontend

The Mysoft Neo UI is a high-performance Angular 17 single-page application using OnPush change detection and lazy-loaded feature modules. The room allocation grid renders up to 500 rooms in a virtual scroll canvas with zero layout jank. Azure SignalR Service pushes real-time housekeeping status, reservation arrivals, and POS order states to every connected terminal simultaneously — no polling, no stale data.

  • Virtual scroll for 500+ room matrices
  • Real-time push via Azure SignalR Service
  • PWA-ready for tablet front desk usage
  • Role-based UI — Receptionist / Manager / Owner
Azure SQL · EF Core · Geo-Replication

Enterprise Data Integrity via Azure SQL

Every monetary transaction — room charges, F&B bills, banquet invoices, payroll journals — is persisted in Azure SQL Database with full ACID compliance. Row-level locking and snapshot isolation prevent dirty reads across simultaneous front-desk and dining-hall transactions. Active geo-replication ensures a readable secondary replica with <5s lag, and automated backups with point-in-time restore safeguard against data loss events.

  • ACID transactions across PMS + POS + Payroll
  • Active geo-replication for DR readiness
  • Tenant-isolated schemas for data sovereignty
  • Query performance insights via Azure Monitor
